可以改，而且**不要在 `TestRunner` 里简单循环调用 10 次 `measureResistance()` / `measureCapacitance()`**。

因为你现在的 `SCPIController::measureCapacitance()` 里面已经包含：

```cpp
CONF:CAP
等待 1000 ms
READ?
```

如果外面循环 10 次，就会变成：

```text
CONF:CAP + 等1s + READ?
CONF:CAP + 等1s + READ?
...
```

总时间会变成 10 秒以上，不是你想要的“1 秒内读 10 次”。

正确做法是：**SCPIController 里新增“配置一次 + 1 秒窗口内多次 READ?” 的函数。**

---

## 1. 会不会阻塞 SCPI 通道？

会占用 SCPI 通道，但这是正常的。

SCPI 本来就是典型的：

```text
发送命令
等待响应
再发送下一条命令
```

同一个万用表连接上，不应该多个线程同时读写。

你现在阻容测试是在 `TestRunner` 的 `QThread` 里跑，所以：

```text
会阻塞阻容测试线程
不会阻塞主界面 UI
不会影响动态 CAN，因为动态是在阻容结束后才开始
```

真正要注意的是：**单次 `READ?` 的 timeout 不能还是 10000 ms**。
否则如果某一次 `READ?` 没响应，可能一下卡 10 秒。

建议：

```cpp
1秒窗口内读10次
每次 READ? timeout 设 150~200 ms
两次读取间隔 100 ms 左右
```

---

## 2. 修改思路

现在流程：

```text
CONF:RES / CONF:CAP
等待 1s
READ?
判断一次
```

改成：

```text
CONF:RES / CONF:CAP
启动 1s 计时窗口
循环最多 10 次：
    READ?
    解析数值
    判断是否在标准范围内
    如果有一次在范围内，立即 PASS 并退出
1s 到了仍无合格值，则 NG，显示最后一次有效值
```

```cpp
double SCPIController::readValueOnce(const QString &readCommand,
                                     int timeoutMs,
                                     QString *rawResponse)
{
    QString response;

    if (!query(readCommand, &response, timeoutMs)) {
        return qQNaN();
    }

    if (rawResponse) {
        *rawResponse = response;
    }

    bool ok = false;
    const double value = parseDoubleResponse(response, &ok);

    if (!ok) {
        setLastError(QString("SCPI响应无法解析：%1").arg(response));
        return qQNaN();
    }

    return value;
}
```

## 3. 增加 1 秒窗口多次读取函数

继续在 `scpi_controller.cpp` 里加：

```cpp
double SCPIController::measureWithPassWindow(const QString &confCommand,
                                             int windowMs,
                                             int sampleCount,
                                             double minValue,
                                             double maxValue,
                                             bool *pass,
                                             QString *sampleText)
{
    clearLastError();

    if (pass) {
        *pass = false;
    }

    if (sampleText) {
        sampleText->clear();
    }

    if (sampleCount <= 0) {
        sampleCount = 1;
    }

    if (windowMs <= 0) {
        windowMs = 1000;
    }

    if (m_simulationMode) {
        const double simulatedValue =
                confCommand.contains("CAP", Qt::CaseInsensitive)
                ? 2.2e-9
                : 10000.0;

        if (pass) {
            *pass = simulatedValue >= minValue && simulatedValue <= maxValue;
        }

        if (sampleText) {
            *sampleText = QString::number(simulatedValue, 'g', 12);
        }

        return simulatedValue;
    }

    /*
     * 关键：这里只配置一次，不在每次 READ? 前重复 CONF。
     */
    if (!configureMeasurementMode(confCommand,
                                  0,
                                  SCPI_WRITE_TIMEOUT_MS)) {
        return qQNaN();
    }

    QElapsedTimer timer;
    timer.start();

    const int intervalMs = qMax(1, windowMs / sampleCount);

    /*
     * 单次 READ? timeout 不能太长，否则一次异常就卡住很久。
     * 建议 150~200 ms。
     */
    const int readTimeoutMs = qBound(50, intervalMs + 50, 200);

    double lastValidValue = qQNaN();
    double firstPassValue = qQNaN();

    QStringList samples;

    for (int i = 0; i < sampleCount && timer.elapsed() < windowMs; ++i) {
        QString raw;
        const double value = readValueOnce(SCPI_READ, readTimeoutMs, &raw);

        if (std::isfinite(value)) {
            lastValidValue = value;

            const bool currentPass = value >= minValue && value <= maxValue;

            samples << QString("#%1=%2%3")
                       .arg(i + 1)
                       .arg(value, 0, 'g', 12)
                       .arg(currentPass ? "(PASS)" : "(NG)");

            if (currentPass) {
                firstPassValue = value;

                if (pass) {
                    *pass = true;
                }

                break;
            }
        } else {
            samples << QString("#%1=无效").arg(i + 1);
        }

        const qint64 nextTime = static_cast<qint64>(i + 1) * intervalMs;
        const qint64 sleepMs = nextTime - timer.elapsed();

        if (sleepMs > 0 && i + 1 < sampleCount) {
            QThread::msleep(static_cast<unsigned long>(sleepMs));
        }
    }

    if (sampleText) {
        *sampleText = samples.join("；");
    }

    if (std::isfinite(firstPassValue)) {
        return firstPassValue;
    }

    return lastValidValue;
}
```

## 5. 实现 measureResistanceWithWindow()

替换原来的 `measureResistance()`：

```cpp
bool TestRunner::measureResistanceWithWindow(double minKOhm,
                                             double maxKOhm,
                                             double &resistanceKOhm,
                                             bool &pass,
                                             QString &sampleText,
                                             QString &errorMessage)
{
    resistanceKOhm = -1.0;
    pass = false;
    sampleText.clear();
    errorMessage.clear();

    if (!m_dmm) {
        errorMessage = "万用表/电桥对象为空";
        return false;
    }

    bool passOhm = false;

    const double resistanceOhm =
            m_dmm->measureResistanceWithPassWindow(STATIC_METER_WINDOW_MS,
                                                   STATIC_METER_SAMPLE_COUNT,
                                                   minKOhm * 1000.0,
                                                   maxKOhm * 1000.0,
                                                   &passOhm,
                                                   &sampleText);

    if (resistanceOhm < 0.0 || !std::isfinite(resistanceOhm)) {
        errorMessage = QString("电阻返回值无效，采样=%1").arg(sampleText);
        return false;
    }

    resistanceKOhm = resistanceOhm / 1000.0;
    pass = passOhm;

    return true;
}
```

# 三、为什么不能直接在 TestRunner 里循环 10 次？

不要这么写：

```cpp
for (int i = 0; i < 10; ++i) {
    measureCapacitance(...);
}
```

因为你现在的 `measureCapacitance()` 内部已经是：

```text
清缓存
CONF:CAP
等待 1000 ms
READ?
```

循环 10 次会变成：

```text
10 次 CONF
10 次 1s 等待
10 次 READ?
```

这会导致：

```text
单个电容测试点至少 10 秒
一整套阻容测试时间暴增
```

---

# 四、这个方案的阻塞风险

这个方案会阻塞当前阻容测试线程约 1 秒，这是正常的。

不会阻塞主界面，因为自动阻容测试在 `QThread` 里。

但是要注意两个点：

## 1. 单次 READ? timeout 不要太长

建议：

```cpp
const int readTimeoutMs = qBound(50, intervalMs + 50, 200);
```

如果 1 秒读 10 次，`intervalMs` 大概是 100 ms，所以单次 timeout 约 150 ms。

不要用：

```cpp
10000 ms
```

否则某次无响应会卡住。

## 2. 不要并发访问同一个 SCPIController

手动阻容窗口、自动阻容线程不要同时读万用表。你现在主流程应该已经有：

```text
自动化测试中禁用手动测试入口
```

这样就没问题。

---

## 五、实际测试建议

先不要一上来就 10 次全开，可以先用：

```cpp
STATIC_METER_SAMPLE_COUNT = 5;
STATIC_METER_WINDOW_MS = 1000;
```

确认仪器响应速度后，再改成：

```cpp
STATIC_METER_SAMPLE_COUNT = 10;
```

如果日志里经常出现：

```text
#1=无效；#2=无效；#3=无效
```

说明仪器一次 `READ?` 响应时间超过 150~200 ms，这时候应该改成：

```cpp
sampleCount = 5
readTimeoutMs = 250
```

会比强行 10 次稳定。

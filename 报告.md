# 阻容与动态一体化测试系统开发报告

## 一、项目背景与需求分析

### 1.1 项目背景

电涡流传感器作为关键检测类产品，其出厂测试过程需要同时覆盖静态阻容测试和动态性能测试。原有测试流程中，阻容测试与动态测试分别布置在两个独立工位，产品需要先完成阻容测试，再流转至动态测试工位进行动态检测。该模式虽然功能上能够满足测试需求，但在实际生产过程中存在测试链路分散、人工操作步骤多、数据追溯不集中、工位节拍较长等问题。

原阻容测试工位节拍为 **52.21 s**，动态测试工位节拍为 **74.29 s**。若按完整测试链路计算，单件产品完成阻容与动态测试总节拍为：

```text
52.21 s + 74.29 s = 126.50 s
```

工位合并后，阻容与动态一体化自动测试节拍为 **96.60 s**，相比原分离测试总节拍减少：

```text
126.50 s - 96.60 s = 29.90 s
```

节拍优化比例约为：

```text
29.90 / 126.50 ≈ 23.64%
```

因此，本次开发的核心目标是将原先分散的阻容测试与动态测试流程整合到同一测试台架和同一套上位机系统中，实现扫码、阻容检测、动态检测、结果判定、数据保存、统计追溯的一体化自动闭环。

### 1.2 现有流程存在的问题

原测试模式主要存在以下问题：

第一，测试工位分散。阻容测试和动态测试分别由不同工位完成，产品需要在两个工位之间流转，增加了搬运、扫码、确认和人工操作时间。

第二，测试数据分散。阻容测试结果和动态测试结果分别保存，缺少以单个产品条码为核心的完整测试结果记录，不利于后续质量追溯。

第三，人工操作风险较高。两个工位分别扫码和测试，存在错扫、漏扫、重复测试、测试结果与产品条码对应错误等风险。

第四，节拍存在冗余。两个独立工位之间存在重复扫码、设备等待、产品上下料、人工确认等环节，导致完整测试链路节拍较长。

第五，异常定位效率不足。阻容异常、动态异常以及工位状态信息分散，现场问题需要分别查看不同测试记录和日志，不利于快速定位。

### 1.3 开发需求

本次开发需求主要包括以下几个方面。

#### 1.3.1 测试流程一体化

系统需要支持在同一测试台架内完成阻容测试和动态测试。操作员完成产品扫码后，上位机自动执行阻容测试流程；阻容测试完成后，系统自动切换硬件线路，进入动态测试流程；动态测试完成后，系统自动输出最终 PASS/NG 结果。

完整流程为：

```text
扫码
→ 阻容测试
→ 阻容结果判定
→ 线路切换
→ 动态测试
→ 动态结果判定
→ 最终结果显示
→ 数据保存
→ 每日统计更新
```

#### 1.3.2 单产品与双产品模式兼容

考虑现场存在单圈测试和双圈测试需求，系统需要兼容位置1单产品测试以及位置1、位置2双产品测试。

当位置2条码为空时，系统进入单产品模式，仅测试位置1产品；当位置1和位置2均输入有效条码时，系统进入双产品模式，分别记录两个位置的阻容测试结果和动态测试结果。

#### 1.3.3 硬件状态自动切换

系统需要通过 PLC 控制测试台架完成以下动作：

```text
阻容测试总使能
位置1/位置2测试回路选择
阻容测试通道复位
阻容线路与动态线路切换
伺服电源开关控制
```

上位机需要根据当前测试阶段自动控制 PLC 点位，避免人工切换线路或手动控制电源。

#### 1.3.4 阻容测试稳定性提升

阻容测试由万用表通过 SCPI 指令完成电阻和电容测量。现场调试中发现，万用表返回数据存在偶发波动，若单次读取结果异常，容易导致误判。因此需要优化测量策略，由原先“等待 1 s 后单次读取”优化为“1 s 测量窗口内多次读取，只要存在一次测量值满足标准即判定合格”，提高测试抗干扰能力。

#### 1.3.5 结果显示与数据追溯

系统需要在主界面实时显示阻容测试结果和动态测试结果，并在测试完成后显示最终 PASS/NG 结果。同时需要自动保存以下数据：

```text
阻容测试明细结果
动态测试明细结果
自动化测试完整结果
每日测试统计结果
```

其中自动化测试完整结果需要以产品条码为核心，记录开始时间、结束时间、产品编码、最终结果、阻容结果、动态结果、位置以及各测试项实测值和标准，便于后续追溯。

## 二、硬件改制方案

### 2.1 硬件改制目标

本次硬件改制的目标是将原本分离的阻容测试工位和动态测试工位集成到同一台架中，使同一产品在同一工位内完成完整测试流程。改制后，操作员无需在两个工位之间转移产品，也无需人工切换测试线路，系统通过 PLC 和上位机自动完成阻容线路、动态线路、测试位置和伺服电源的切换控制。

硬件改制主要围绕以下目标展开：

```text
实现阻容测试线路与动态测试线路共台架集成
实现位置1和位置2测试回路选择
实现阻容通道复位
实现动态测试伺服电源控制
实现上位机对 PLC 点位的自动控制
```

### 2.2 阻容测试线路与动态测试线路切换

阻容测试和动态测试对产品接入线路的要求不同。阻容测试阶段需要将产品接入万用表测量回路，用于检测不同信号线之间的电阻和电容；动态测试阶段需要将产品接入动态测试回路，通过 CAN 通讯采集动态检测数据。

因此，台架硬件侧增加了阻容测试线路与动态测试线路切换功能，由 PLC 控制继电器实现线路切换。

对应 PLC 控制点为：

```text
M26：动态/阻容线路切换
M26 OFF：阻容测试线路
M26 ON ：动态测试线路
```

测试开始时，系统先控制 M26 关闭，使产品接入阻容测试线路；阻容测试完成后，系统控制 M26 打开，将产品切换至动态测试线路。

### 2.3 位置1/位置2测试回路选择

由于台架需要支持双产品测试，阻容测试阶段需要选择当前测量的是位置1产品还是位置2产品。硬件侧通过 PLC 控制位置选择继电器，实现万用表测量回路在位置1和位置2之间切换。

对应 PLC 控制点为：

```text
M20：阻容测试位置选择
```

现场调试过程中，通过短接位置2产品的 `sin+` 与 `sin-` 进行验证，发现初始软件逻辑与实际台架接线关系相反。表现为实际短接位置2，但阻容测试异常显示在位置1；而动态测试能够正确显示位置2 NG、位置1 PASS。

该现象说明动态测试通道映射正确，阻容测试位置选择逻辑与实际硬件接线相反。最终将阻容测试位置选择逻辑修正为：

```cpp
m_plc->setMRelay(M_POSITION_SELECT, position == 1);
```

即实际控制关系为：

```text
M20 ON  = 位置1
M20 OFF = 位置2
```

修正后，阻容测试结果归属与实际物理位置保持一致。

### 2.4 阻容测试总使能

阻容测试阶段需要打开阻容测试总使能，使测试继电器矩阵或测量回路进入有效状态。该功能由 PLC M 点控制。

对应控制点为：

```text
M19：阻容测试总使能
```

该点属于保持型信号。阻容测试开始前，根据测试流程打开阻容使能；阻容测试完成后，关闭阻容使能，避免阻容测试回路对后续动态测试产生影响。

### 2.5 阻容通道复位

阻容测试涉及多个测试项和多组继电器通道切换。为避免上一测试项继电器状态残留，影响下一测试项测量结果，系统在测试模式切换和关键流程节点增加阻容通道复位动作。

对应控制点为：

```text
M100：阻容测试通道复位
```

M100 属于触发型信号，不应长时间保持 ON。因此程序中采用脉冲控制方式：

```cpp
pulseMRelay(M_RESET_CHANNEL, 100);
```

其动作含义为：

```text
M100 ON
延时 100 ms
M100 OFF
```

这样可以触发 PLC 完成一次阻容通道复位，同时避免 M100 长时间保持导致测试通道一直处于复位状态。

### 2.6 伺服电源控制

动态测试阶段需要打开伺服电源，而阻容测试阶段不应打开伺服电源，否则可能影响万用表测量结果和测试安全。因此台架增加了伺服电源控制逻辑。

对应 PLC 控制点为：

```text
M27：伺服电源
M27 OFF：伺服关闭
M27 ON ：伺服打开
```

测试流程中，阻容测试准备阶段先关闭伺服电源；阻容测试完成并切换至动态测试线路后，再打开伺服电源，进入动态测试流程；动态测试完成后，再关闭伺服电源并恢复安全状态。

### 2.7 PLC 通讯方式调整

现场 PLC 为三菱 FX3U，使用 USB-SC09 编程口通讯，不采用 Modbus 协议。因此本次上位机 PLC 通讯逻辑采用 FX 系列编程口协议。

PLC 通讯参数如下：

```text
PLC型号：三菱 FX3U
通讯线：USB-SC09
通讯方式：FX 编程口协议
波特率：9600
数据位：7
校验位：Even
停止位：1
流控：None
```

程序控制方式由原先的 Y 点控制调整为 M 点控制，统一使用：

```cpp
setMRelay(...)
pulseMRelay(...)
```

其中：

```text
setMRelay(...)：用于保持型 ON/OFF 控制
pulseMRelay(...)：用于脉冲型触发动作
```

本项目主要使用点位如下：

| PLC点位 | 功能        | 控制方式   |
| ----- | --------- | ------ |
| M19   | 阻容测试总使能   | ON/OFF |
| M20   | 阻容测试位置选择  | ON/OFF |
| M26   | 动态/阻容线路切换 | ON/OFF |
| M27   | 伺服电源      | ON/OFF |
| M100  | 阻容测试通道复位  | 脉冲     |

## 三、技术路线

### 3.1 总体技术路线

本系统采用 Qt C++ 开发上位机软件，通过串口与 PLC 通讯，通过 VISA/SCPI 与万用表通讯，通过 CAN 与动态测试设备通讯，实现阻容测试与动态测试的一体化自动控制。

系统总体技术路线如下：

```text
Qt 上位机
    ↓
扫码录入产品条码
    ↓
PLC 控制台架进入阻容测试模式
    ↓
万用表 SCPI 执行电阻/电容测量
    ↓
依据 INI 测试标准判定阻容结果
    ↓
PLC 切换至动态测试模式
    ↓
CAN 通讯执行动态测试
    ↓
依据动态标准判定动态结果
    ↓
生成最终 PASS/NG
    ↓
保存阻容结果、动态结果、完整结果和每日统计
```

### 3.2 软件架构设计

软件采用模块化设计，主要包括以下模块：

| 模块                    | 功能                                         |
| --------------------- | ------------------------------------------ |
| MainWindow            | 主界面、自动化流程调度、结果显示、数据保存                      |
| TestRunner            | 阻容测试执行线程，负责测试项循环、PLC 点位控制、万用表测量            |
| SCPIController        | 万用表通讯，执行 `CONF:RES`、`CONF:CAP`、`READ?` 等指令 |
| PLCController         | FX3U PLC 通讯，负责 M 点 ON/OFF 和脉冲控制            |
| DynamicTestController | 动态测试控制，负责 CAN 通讯、动态数据解析和结果判定               |
| CsvExporter           | 阻容测试结果导出                                   |
| DynamicCsvExporter    | 动态测试结果导出                                   |
| DailyTestStatistics   | 每日测试统计、重复条码识别、统计文件保存                       |
| 配置解析模块                | 从 INI 文件读取阻容和动态测试标准                        |

该架构将界面显示、测试流程、设备通讯、结果保存进行解耦，便于后续维护和功能扩展。

### 3.3 自动化测试流程设计

自动化测试流程由主界面统一调度，分为阻容测试阶段和动态测试阶段。

#### 3.3.1 阻容测试准备

自动化测试开始后，系统首先进入阻容测试准备阶段：

```text
关闭伺服电源 M27 OFF
切换至阻容线路 M26 OFF
关闭阻容测试总使能 M19 OFF
触发阻容通道复位 M100 脉冲
```

该阶段用于确保台架处于安全、稳定的阻容测试状态。

#### 3.3.2 阻容测试执行

阻容测试由 `TestRunner` 在线程中执行。测试流程包括：

```text
读取 INI 中阻容测试项
选择测试位置
设置 PLC 测试点位
配置万用表测量模式
读取电阻/电容值
与测试标准比较
生成单项测试结果
```

阻容测试项包括：

```text
GND -> sin+
GND -> sin-
GND -> cos+
GND -> cos-
sin+ -> sin-
cos+ -> cos-
5V -> GND
```

其中，GND 相关测试项执行指定方向测量，绕组类测试项执行红黑方向测量，`5V -> GND` 仅测试电容，不测试电阻。

#### 3.3.3 动态测试准备

阻容测试完成后，系统自动切换至动态测试模式：

```text
关闭阻容测试总使能 M19 OFF
触发阻容通道复位 M100 脉冲
切换至动态线路 M26 ON
延时等待线路稳定
打开伺服电源 M27 ON
```

随后启动动态 CAN 测试。

#### 3.3.4 动态测试执行

动态测试由 `DynamicTestController` 执行，通过 CAN 发送复位帧、启动帧，并接收动态测试数据。系统根据动态配置中的测试项、单位和标准进行结果判定，最终生成位置1和位置2对应的动态测试结果。

#### 3.3.5 测试完成处理

动态测试完成后，系统执行收尾动作：

```text
关闭伺服电源 M27 OFF
切回阻容线路 M26 OFF
生成最终结果
显示 PASS/NG 图片
保存测试数据
更新每日统计
准备下一次扫码
```

### 3.4 万用表 SCPI 测量策略

阻容测试采用万用表 SCPI 指令完成电阻和电容测量。原测量策略为：

```text
发送 CONF:RES 或 CONF:CAP
等待 1 s
发送 READ?
读取一次结果
```

现场调试发现，万用表偶发返回值波动较大，单次读取容易造成误判。因此优化为：

```text
发送 CONF:RES 或 CONF:CAP
在 1 s 窗口内多次发送 READ?
只要任意一次读取值满足标准，即判定该项合格
若全部读取值均不满足标准，则判定 NG
```

该策略在不显著增加测试节拍的前提下，提高了阻容测试对瞬时波动的容忍度。

同时，为避免 SCPI 通道长时间阻塞，单次 `READ?` 不使用原 10 s 超时，而是采用较短超时时间，使 1 s 窗口内能够完成多次读取。

### 3.5 数据保存技术路线

系统测试完成后，自动保存多类数据。

#### 3.5.1 阻容测试结果

阻容测试结果由 `CsvExporter` 保存，记录每个测试项的实测电阻、实测电容、标准、结果和异常信息。

#### 3.5.2 动态测试结果

动态测试结果由 `DynamicCsvExporter` 保存，记录每个动态测试项的测试值、单位、标准和判定结果。

#### 3.5.3 自动化测试完整结果

自动化测试完整结果用于将阻
